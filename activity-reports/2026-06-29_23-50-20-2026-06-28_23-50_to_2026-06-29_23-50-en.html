Sabre

  • Sabre added/fixed custom VObject property type registration in pull/402, related to frontend retro-compat issue issues/1093 and integration-test update pull/280.

Frontend

  • Booking API work started in pull/1094: shared booking DTOs, private authenticated booking-link DAO, public booking DAO, and public booking route/page scaffold.
  • Review found two functional blockers in pull/1094:
    • The public /booking/:bookingLinkPublicId route currently resolves to a page that renders null and hides failures; do not expose it before basic loading/error UI or a feature gate is added. See pull/1094 review comment.
    • Slot fetching uses local midnights and an inclusive-looking end date that excludes the last day after midnight; it should query a UTC month window with an exclusive upper bound. See pull/1094 review comment.
  • The jCal retro-compat analysis in issues/1093 concluded:
    • Read path is already type-agnostic.
    • Write path still emits unknown, but backend accepts it.
    • Series metadata diff may falsely detect changes because it compares the full jCal tuple, including type and empty-value representation.
    • Final direction in the issue was that retro-compatibility should be handled on the Sabre side.
  • Frontend multiple alarms/attendee support pull/1091 was approved after CodeScene improved status; Docker images were published.
  • Frontend organizer asJcal update pull/1075 received only low-signal maintainability feedback and Docker image publication.
