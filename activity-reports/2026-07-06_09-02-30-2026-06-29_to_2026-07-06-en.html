TL;DR

  • Booking links progressed across backend and frontend: public booking APIs/UI landed, slots now expose owner/details/timezone support, cancel/get/reservation token routes were merged, and acknowledgement email enrichment is still open in pull/906.
  • Reindex/search reliability received major fixes: Mongo cursor timeout was fixed and deployed via pull/891, Sabre filterless REPORT performance was improved in pull/406, but recurring-event indexing correctness remains an important open item in issues/895 / pull/898.

Highlights

Backend / Side Service / Sabre

Booking links

  • Side Service merged several booking-link backend pieces:
    • Token signing and reservation JWT flow via pull/873 and pull/874.
    • Public slots now expose owner/details and booking metadata via pull/878, with requested timezone support in pull/880.
    • Admin deletion of booking-link-created events was merged in pull/876; an unstable offset paging bug was caught during review and fixed by using DAV listing as source of truth instead of paging the search index.
    • Booked event cancel/get routes were merged in pull/881 and pull/886. Review clarified idempotency/error handling for cancel and that the get endpoint can rely on the public slots endpoint when the public id is available.
  • Booking confirmation email still needs follow-through: issues/899 remains open and pull/906 is open, with approval signal but not merged in the data.
  • Frontend API wiring and UI consumed the new booking APIs through pull/1094 and pull/1097. Follow-ups include attendee timezone selection issues/1103, open slot filtering work pull/1104, and better validation/error handling captured in review on pull/1097.

Address book / CardDAV webadmin

  • WebAdmin task visibility improved: effective running options are now exposed in task details via pull/890 / issues/889.
  • Domain provisioning was tightened:
    • Only mail DNS config is required for domain provisioning in pull/893.
    • Provisioned SaaS users are registered as Domain Members regardless of user search mode in issues/904 / pull/905.
  • Release/deployment notes show Side Service images deployed around 2.4.1.x, including reindex fixes and the OpenSearch index switch to calendar-4 for shared/delegated calendar search in issues/871.

Team Calendar

  • Side Service merged Team Calendar WebAdmin CRUD APIs in pull/901. Review clarified behavior: create is not idempotent, delete should be idempotent, and tests were added to make that explicit.
  • Team Calendar AMQP list notifications are ignored in pull/902, closing issues/892.
  • Sabre Team Calendar support was merged in pull/407, backed by integration contract tests in pull/283.
  • A remaining visibility/security follow-up was explicitly created: private/confidential events in public team calendars need coverage in issues/289.
  • There is still a product/search requirement signal: team calendar name is needed for quick search, despite an initial question about whether the ticket was needed in issues/834.

Reindex / search / Sabre performance

  • Side Service fixed large reindex failures caused by Mongo cursor timeout by materializing user/resource listings before slow per-item processing in pull/891, closing issues/887.
  • Sabre fixed the root performance issue for filterless XML calendar-query REPORT by batching / single-querying data in pull/406, closing issues/403. Preprod signal showed large improvement, but reindex still reported 4 failed events due to malformed ICS.
  • Malformed ICS with observance-less VTIMEZONE was analyzed in issues/896. A sanitizer attempt pull/897 was considered too hacky; final direction appears to ignore the affected test events rather than ship parser workaround.
  • Recurring event indexing race remains the most important unresolved backend correctness topic: issues/895 identified delete-then-index race and stale overridden occurrence resurrection; pull/898 evolved to use UID collapse, stale occurrence pruning, indexed sequence, stored recurrenceId, and upgrade instructions, but is still open in the report.

Sabre ACL / scheduling / release

  • Sabre release 2.4.1 was deployed with SABRE_EMAIL_VALARM_RECIPIENT_SCHEDULING=true and CALDAV_ORGANIZER_VALIDATION=true, then organizer validation was temporarily disabled due to attendee PARTSTAT regression, tracked in issues/408.
  • Attendee PARTSTAT update 403 was fixed in pull/410, with integration coverage in pull/285.
  • Read ACL enforcement for PROPFIND and JSON reports was fixed in pull/411, with integration tests in pull/286, pull/287, and pull/288.
  • Duplicate DAV / X-Sabre-Version response headers were fixed in pull/405, closing issues/404.
  • VALARM merge logic remains open in issues/409, with related frontend alarm issue issues/1100.

Frontend

  • Public booking flow:
    • Booking DAO/API calls and public route scaffolding landed in pull/1094.
    • Basic booking UI and flow were merged in pull/1097. QA caught and addressed locale/time display, cut-off calendar row, owner readability, backend error display, form reset, stale slots, and timezone request handling.
    • pull/1104 is open to filter unavailable/past dates and slots; CodeRabbit found a real empty-state bug when all visible slots are filtered out.
  • Deep links:
    • /events/:uid support was merged in pull/1102. QA found the UID REPORT response includes VTIMEZONE; extractCalendarEvents was fixed to skip non-vevent components.
    • /newEvent?attendee=... prefill support is open in pull/1106, with QA saying it works and README route documentation added. Direct component test coverage was suggested but appears low-risk.
  • Event creation regression:
    • issues/1095 / pull/1096 fixed organizer.asJcal is not a function by ensuring organiserData remains a full userOrganiser instance in Redux state.
  • Platform/mail URL:
    • pull/1099 added URI template support for MAIL_SPA_URL and reused the shared resolver for visio templates. A review suggestion to encodeURIComponent substitutions was explicitly rejected because placeholders are used in host/subdomain positions where percent encoding would break DNS labels.
  • Booking-link ICS retro-compat:
    • issues/1093 concluded read parsing already ignores jCal value type; retro-compat responsibility moved to Sabre side, with related Sabre normalization in pull/402.

Integration Tests / QA

  • Test images were bumped for Sabre and Side Service via pull/281 and pull/290.
