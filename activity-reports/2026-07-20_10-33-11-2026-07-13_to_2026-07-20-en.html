Highlights

Public agenda / booking-link email cancellation

  • pull/953 removed ICS attachments from booking-link-related emails after issues with mail clients rendering incorrect or misleading invitation content. The team agreed to rely on a better email body instead.
  • Sabre metadata preservation was fixed through pull/427 and pull/428, then extended to X-PUBLICLY-CANCELLED-BY in pull/430.
  • The cancellation fix plan for issues/952 converged on adding X-PUBLICLY-CANCELLED-BY, using a dedicated booking-cancelled email template, and updating organizer events with STATUS:CANCELLED; pull/304 covers the organizer PUT STATUS:CANCELLED behavior.
  • pull/969 is still WIP and has a CodeScene warning on test complexity in BookedEventCancelledTest.java.

Booking links / availability / frontend UX

  • pull/959 merged extraAttendees support for booking links. Review pushed the API and Mongo representation from a flat list to an and tree of participant leaves to keep future optional/substitute attendee logic additive.
  • Frontend booking-link UX improvements landed: private booking management UI pull/1145, websocket updates for booking links pull/1148, edit availability rule fix pull/1153, public page empty-name spacing pull/1158, and date hover state pull/1159.
  • Availability working days now use ISO-8601 format via pull/1144. A valid review finding led to guarding malformed daysOfWeek payloads with Array.isArray to avoid reducer hydration crashes.
  • Regular hours were implemented and refined in pull/1137, with the display default aligned to 9:00–18:00 after product feedback in issues/1149.

Team calendar scheduling / Sabre

  • Sabre team calendar scheduling work progressed in pull/424, including stronger validation that a given team calendar contains the relevant event UID before resolving replies.
  • Organizer transfer to write members was merged in pull/429, backed by integration coverage in pull/303.
  • A security-related integration-test signal was raised for team calendar ITIP: TEAM_CALENDAR_ID must not bypass ACLs when a recipient lacks access, captured in the discussion on pull/300.

Auth / provisioning

  • Sabre auto-provisioning on DAV request was merged in pull/426. Review required provisioning only when the user exists in LDAP and populating names from LDAP rather than hardcoding empty values.
  • There is an architectural tradeoff noted on pull/426: direct MongoDB writes avoid cross-service calls on the auth path but couple Sabre to side-service storage schema.

Frontend contact card / cross-app integrations

  • pull/1150 adds attendee contact cards but remains open. Review explicitly asked not to expose chat unless the action is scoped and ready, and to gate mail action on Mail SPA configuration.
  • Chat deep-linking is not fully specified. A web-only workaround was proposed in issues/1086 comment, but open questions remain around how Calendar obtains Matrix IDs and what to do for non-Twake users.
  • File picker / Drive integration was scoped into follow-up work in issues/247: iframe rendering behind an env gate, postMessage communication, then refinement.
