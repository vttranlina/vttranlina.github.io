  • comment on #898 — @chibenwa, 2026-07-04 04:20

    Fixed the failing searchShouldReturnOverriddenOccurrenceWhenOnlyItMatches test (OpenSearch backend, build #6).

    Root cause: with the collapse-on-uid approach, a keyword that only matches an overridden occurrence (e.g. Bob invited on a single instance) correctly surfaces that occurrence rather than the master. But the OpenSearch CalendarEventsDocument did not store recurrenceId, so the serialize/deserialize round-trip dropped it — the returned occurrence came back with recurrenceId=Optional.empty instead of Optional[2025-01-03T10:00:00Z], which is exactly what the assertion reported. The Memory backend was unaffected because it keeps the EventFields objects as-is.

    Fix (commit f6fa09f3): store recurrenceId as a non-indexed keyword (same treatment as resourceName) in both CalendarEventsDocument and the index mapping, so an overridden occurrence surfaced by search keeps its recurrenceId.
